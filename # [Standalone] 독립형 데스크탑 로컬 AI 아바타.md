# [Standalone] 독립형 데스크탑 로컬 AI 아바타 개발 명세서 (최종본)

## 1. 프로젝트 개요
본 프로젝트는 특정 프로그램에 종속되지 않고 **단독 실행(.py 또는 .exe)**되는 바탕화면 상주형 AI 아바타(Desktop Pet) 애플리케이션이다. 투명한 배경 위에서 아바타가 스스로 움직이며, 사용자와 로컬 AI(Ollama) 간의 소통을 돕고 마우스 조작을 통해 크기 및 위치를 자유롭게 제어하는 것을 목표로 한다.

## 2. 하드웨어 및 소프트웨어 환경
* **CPU:** AMD Ryzen 7 5800H
* **GPU:** NVIDIA GeForce RTX 3070 (VRAM 8GB)
* **RAM:** 64GB
* **AI 백엔드:** Ollama API (localhost:11434 / 모델: gemma-2-2b-it-gguf 등)
* **개발 언어 및 라이브러리:** Python 3.x, PyQt5, Requests

## 3. 핵심 기능 요구사항

### 3.1. 아바타 시각 구현 (Transparency & Format)
* **배경 투명화:** 아바타 이미지를 제외한 나머지 윈도우 영역은 완벽하게 투명해야 하며, 바탕화면 아이콘이나 다른 프로그램 창 위에 캐릭터만 떠 있는 것처럼 보여야 한다.
* **사용 포맷:** 투명 배경(Alpha Channel)이 포함된 **GIF** 또는 **WebP** 형식을 사용하여 끊김 없는 애니메이션을 구현한다. (AVI 등 일반 영상 포맷은 투명도 처리가 복잡하므로 배제)

### 3.2. 자동 모션 시스템 (State-based Animation)
아바타는 상황에 따라 다음 두 가지 상태를 자동으로 전환하며 움직여야 한다.
1. **대기 상태 (Idle):** 질문이 없을 때 스스로 숨을 쉬거나 눈을 깜빡이는 애니메이션 (`idle.gif` 재생).
2. **응답 상태 (Talk):** Ollama가 답변을 생성하여 텍스트를 출력하는 동안 입을 맞추어 움직이는 애니메이션 (`talk.gif` 재생). 답변 완료 시 다시 대기 상태로 복귀한다.

### 3.3. 인터랙티브 조작 (Mouse Interaction)
* **크기 조절 (Zoom):** 아바타 위에 마우스를 올리고 **휠(Wheel)**을 굴리면 캐릭터의 크기가 실시간으로 커지거나 작아져야 한다. (최소/최대 크기 제한 설정 필요)
* **위치 이동 (Drag):** 마우스 왼쪽 버튼으로 클릭 후 드래그하여 바탕화면 어디든 자유롭게 배치할 수 있어야 한다.
* **항상 위 (Always on Top):** 어떤 작업을 하더라도 아바타가 다른 창 아래로 가려지지 않도록 최상단 노출 상태를 유지한다.
* **동적 대화창 (Hover UI):** 평소에는 아바타만 노출되며, 마우스를 아바타 위로 올렸을 때만 대화 내역과 입력창이 나타나도록 구현한다.

### 3.4. 비동기 통신 및 안정성
* **QThread 활용:** AI 답변을 생성하는 동안 프로그램 전체가 멈추지 않도록(UI Freeze 방지) 반드시 별도의 스레드에서 Ollama와 통신한다.
* **예외 처리:** Ollama가 실행 중이지 않거나 통신 오류 발생 시 사용자에게 텍스트창을 통해 에러 메시지를 노출한다.

## 4. 상세 기술 설정 (PyQt5)
* `Qt.FramelessWindowHint`: 윈도우 테두리 제거
* `Qt.WA_TranslucentBackground`: 배경 투명화 활성화
* `QMovie`: GIF 애니메이션 재생 및 제어
* `wheelEvent`: 마우스 휠을 통한 위젯 리사이징 로직 구현
* `mouseMoveEvent`: 드래그를 통한 좌표 이동 구현
