# [Phase 6] 지능형 함수 호출(Function Calling) 및 자율적 도구 사용 통합 명세서

## 1. 프로젝트 목표
* **핵심 패러다임 전환:** 기존의 '키워드 기반 강제 검색' 방식을 폐기하고, LLM(Ollama)이 스스로 외부 도구(인터넷 검색)가 필요한지 판단하여 호출하는 '함수 호출(Function Calling)' 구조로 전환함.
* **기대 효과:** "링크를 확인하라"는 회피형 답변을 차단하고, 수집된 데이터를 AI가 직접 분석하여 완결된 정보를 제공하는 'Gemini/ChatGPT' 방식의 사용자 경험 구현.

## 2. 시스템 시간 및 컨텍스트 동기화 (필수)
LLM이 시간 감각을 가질 수 있도록 모든 요청의 최상단에 다음 정보를 강제 주입한다.
* **메커니즘:** Python의 `datetime` 라이브러리를 활용하여 프롬프트 전송 직전 생성.
* **주입 내용:** `현재 시각: YYYY-MM-DD (요일) HH:MM`. 
* **지침:** "너는 제공된 현재 시각을 기준으로 모든 질문을 해석해야 한다."

## 3. 도구 정의 (Tool/Function Definition)
Ollama 모델(Qwen 2.5/3.5, Gemma 2 등)이 인식할 수 있도록 다음 함수를 정의하여 시스템 프롬프트에 포함한다.

### 3.1. 함수명: `web_search`
* **설명:** "최신 뉴스, 실시간 날씨, 코인/주식 시세, 인물 정보 등 모델의 학습 데이터에 없는 최신 정보가 필요할 때 호출한다."
* **매개변수:** * `query` (string): 검색 엔진에 전달할 최적화된 검색어.
* **출력 기대값:** 해당 검색어에 대한 웹 사이트의 제목과 상세 본문 요약 텍스트.

## 4. 구동 로직 (Intelligent Loop)

1. **사용자 질문 접수:** (예: "내일 영천 날씨 어때?")
2. **AI 1차 판단 (Tool Choice):** 질문을 분석한 후 도구가 필요하다고 판단되면 답변 대신 특수 형식의 코드(`{"name": "web_search", "arguments": {...}}`)를 반환함.
3. **시스템 도구 실행:** 파이썬이 위 코드를 감지하면 즉시 `duckduckgo-search`를 실행하여 상위 5개 이상의 상세 데이터를 수집함.
4. **결과 피드백 (Tool Output):** 수집된 **날것의 데이터**를 다시 LLM에게 "이것은 네가 요청한 `web_search`의 결과다. 이를 바탕으로 최종 답변을 작성하라"고 전달함.
5. **최종 답변 생성:** AI가 데이터를 분석하여 답변을 생성. 이때 **[Phase 3] 감정 태그**와 **[Phase 4] TTS**를 연동함.

## 5. 지식 추출 및 답변 완결성 규칙
* **회피 금지:** 검색 결과에 링크만 있더라도, 텍스트 내에서 최대한의 수치(기온, 가격 등)를 찾아내어 답변할 것. 절대 "사이트를 방문하라"고 하지 말 것.
* **출처 명시:** 정보를 어디서 찾았는지 간략히 언급하되(예: "기상청 정보에 따르면..."), 답변의 핵심은 구체적인 숫자와 사실 위주로 구성할 것.
* **추론 활용:** 수집된 정보가 단편적일 경우, AI의 추론 능력을 사용하여 사용자에게 가장 도움이 되는 방향으로 재구성할 것.

## 6. 구현 지침 (Python/PyQt5)
* **비동기 처리:** 도구 호출과 검색 과정에서 UI 애니메이션(생각 중 모션)이 끊기지 않도록 `QThread`와 `asyncio`를 정밀하게 설계할 것.
* **성능 최적화:** RTX 3070의 VRAM을 효율적으로 쓰기 위해, Function Calling 능력이 검증된 모델(Qwen 계열 권장)을 사용하고 컨텍스트 길이를 적절히 제한할 것.
