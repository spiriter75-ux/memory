# [Phase 2] 데스크탑 로컬 AI 아바타 UI/UX 고도화 명세서

## 1. 프로젝트 현황 및 목표
* **현재 상태:** PyQt5를 활용하여 테두리가 없는 투명 윈도우 위젯 위에 정적 이미지(아바타)와 채팅창이 렌더링되며, 백그라운드 스레드에서 Ollama API(예: qwen3.5-uncensored:9b)와 성공적으로 통신 중임.
* **Phase 2 목표:** 1. 화면 가림을 최소화하기 위해 '마우스 호버(Hover)' 기반의 동적 채팅창 숨김/노출 기능 구현.
  2. 정적 이미지를 제거하고, AI의 상태(대기/답변 중)에 따라 2개의 애니메이션(WebP 또는 GIF)이 교차 재생되는 상태 기반 렌더링(State-based Rendering) 구현.

## 2. 기능 상세 요구사항

### 2.1. 마우스 호버(Hover) 동적 채팅창
* **기본 상태:** 프로그램 실행 시 채팅 내역 창(`chat_display`)과 텍스트 입력 창(`input_field`)은 숨김(Hide) 처리되어 바탕화면에는 아바타 애니메이션만 보이도록 한다.
* **이벤트 감지:**
  * `enterEvent`: 사용자의 마우스 커서가 아바타(위젯 영역) 위로 올라오면 숨겨져 있던 채팅창과 입력창이 즉시 나타나야(Show) 한다.
  * `leaveEvent`: 마우스 커서가 위젯 영역을 완전히 벗어나면 다시 채팅창과 입력창이 숨겨져야(Hide) 한다.

### 2.2. 상태 기반 애니메이션 전환 (QMovie 활용)
* **미디어 리소스:** `idle.webp` (또는 .gif)와 `talk.webp` (또는 .gif) 두 개의 투명 애니메이션 파일을 사용한다.
* **모드 1: 대기 상태 (Idle Mode)**
  * 기본 상태이며 `idle.webp`를 무한 루프로 재생한다.
* **모드 2: 대화 상태 (Talk Mode)**
  * 사용자가 프롬프트를 입력하고 Ollama API 스레드가 작동을 시작하여 텍스트를 반환하는 동안에는 `idle.webp` 재생을 멈추고 `talk.webp`를 재생한다.
  * 답변 생성이 완전히 종료되면 자동으로 다시 '대기 상태(Idle Mode)'로 복귀한다.

## 3. 구현 지침 (PyQt5)

* **UI 초기화 변경:** `initUI` 메서드 내에서 텍스트 위젯들에 대해 `.hide()` 처리를 기본값으로 적용할 것.
* **애니메이션 처리:** 기존의 `QPixmap`을 사용한 정적 이미지 렌더링을 폐기하고, `QMovie` 객체 2개를 메모리에 로드하여 상태에 따라 `setMovie()`와 `start()`, `stop()` 메서드로 스위칭할 것.
* **성능 최적화:** 애니메이션 프레임 로드로 인한 메인 스레드 프리징이 발생하지 않도록 주의하고, 기존에 구현된 백그라운드 AI 통신 스레드 구조는 그대로 유지할 것.
