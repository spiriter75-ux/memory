# [Phase 4] TTS 음성 합성 및 아바타 립싱크(Lip-Sync) 연동 명세서

## 1. 프로젝트 목표
* **핵심 기능:** Ollama의 텍스트 답변을 고품질 AI 음성으로 변환하여 출력하고, 음성 재생 시간과 아바타의 '말하기(Talk)' 애니메이션을 정밀하게 동기화함.
* **최적화 전략:** 그래픽카드(RTX 3070)의 VRAM을 보호하기 위해 클라우드 기반의 `edge-tts` 라이브러리를 사용하며, 시스템 RAM(64GB)을 활용해 오디오를 처리함.

## 2. TTS 기술 사양 및 보이스 설정

### 2.1. 라이브러리 구성
* **음성 합성:** `edge-tts` (Microsoft Edge AI 엔진 활용)
* **오디오 재생:** `pygame.mixer` 또는 `QtMultimedia` (비동기 재생 지원)

### 2.2. 캐릭터 성격별 보이스 파라미터 (Voice Mapping)
사용자의 선택에 따라 다음 설정을 적용할 수 있도록 변수화한다.
1. **여성 보좌관 (기본):** `ko-KR-SunHiNeural` (Pitch: +0Hz, Rate: +0%)
2. **남성 보좌관:** `ko-KR-InJoonNeural` (Pitch: +0Hz, Rate: +0%)
3. **어린이 보좌관 (커스텀):** `ko-KR-SunHiNeural` (Pitch: **+20Hz**, Rate: **+15%**)
   * *원리: 여성형 보이스의 주파수를 높이고 속도를 약간 빠르게 설정하여 어린아이의 톤을 구현함.*

## 3. 구현 프로세스 (Workflow)

### 3.1. 오디오 생성 및 재생 스레드
1. `OllamaThread`가 답변 텍스트를 생성하면, 즉시 `edge-tts`를 통해 MP3 파일(또는 바이트 스트림)로 변환한다.
2. 음성 파일 생성이 완료되면 오디오 재생을 시작함과 동시에 UI 메인 스레드에 `start_talking` 시그널을 보낸다.

### 3.2. 정밀 립싱크 동기화 (Lip-Sync Logic)
* **시작:** 오디오 재생 시작 시 아바타를 `talk.webp` 애니메이션으로 전환한다.
* **종료:** 오디오 재생이 끝나면(재생 시간 체크 또는 이벤트 감지) 즉시 아바타를 `idle.webp` 상태로 복귀시킨다.
* **주의:** 텍스트가 화면에 찍히는 속도보다 음성 재생이 빠르거나 느릴 수 있으므로, 실제 **오디오 파일의 길이(Duration)**를 기준으로 애니메이션을 제어한다.

## 4. [Phase 3] 감정 인식과의 통합
* **감정 반영:** 감정 태그(`[기쁨]`, `[슬픔]` 등)에 따라 TTS의 **Pitch(음높이)**와 **Rate(속도)**를 미세 조정한다.
    * 예: `[기쁨]` -> Pitch +5Hz, Rate +10% / `[슬픔]` -> Pitch -5Hz, Rate -10%

## 5. 구현 지침 (Python)
* **비동기 처리:** `asyncio`를 활용하여 `edge-tts`의 음성 생성 과정에서 UI가 멈추지 않도록 설계할 것.
* **임시 파일 관리:** 생성된 음성 파일은 재생 후 즉시 삭제하거나 메모리 내 데이터로만 처리하여 디스크 용량 낭비를 방지할 것.
