# [Standalone] 데스크탑 로컬 AI 아바타 개발 명세서

## 1. 프로젝트 개요
본 프로젝트는 특정 프로그램(예: 안티그래비티)에 종속되지 않고 **독립적으로 실행되는 데스크탑 위젯(Desktop Pet) 애플리케이션**이다. 사용자가 PC 환경에서 다른 작업(웹서핑, 문서 작업, 디자인 등)을 하는 동안 바탕화면 최상단에 상주하며, 로컬 AI(Ollama)와 실시간으로 소통할 수 있는 투명한 아바타 인터페이스를 제공하는 것을 목표로 한다.

## 2. 타겟 구동 환경 (Target Environment)
다른 무거운 프로그램과 동시 실행될 수 있으므로, 아래의 시스템 자원 내에서 쾌적하게 구동되도록 최적화한다.
* **CPU:** AMD Ryzen 7 5800H
* **GPU:** NVIDIA GeForce RTX 3070 (VRAM 8GB)
* **RAM:** 64GB
* **백엔드 AI:** Ollama API (localhost:11434, VRAM 점유율을 고려하여 경량 GGUF 모델 통신)
* **프론트엔드 프레임워크:** Python + PyQt5

## 3. 핵심 UI/UX 요구사항 (동작 시나리오)

### 3.1. 완전한 독립 실행
* 본 프로그램은 자체적인 실행 파일(.py 또는 .exe)을 가지며, 클릭 한 번으로 백그라운드 AI(Ollama)와 즉시 연결되어 바탕화면에 나타나야 한다.

### 3.2. 마우스 호버(Hover) 기반 동적 대화창
대화창이 항상 화면을 가리고 있는 것을 방지하기 위해 동적 숨김 처리를 적용한다.
* **기본 상태:** 바탕화면에는 투명한 배경의 아바타 이미지(또는 GIF)만 떠 있다. 텍스트 입력창과 대화 내역은 숨겨져 있다.
* **마우스 진입 (Hover Enter):** 마우스 커서를 아바타 위로 올리면, 아바타 주변에 '채팅 내역 창'과 '텍스트 입력창'이 부드럽게 팝업(Fade-in)된다.
* **마우스 이탈 (Hover Leave):** 마우스가 아바타 및 대화창 영역을 벗어나면 일정 대기 시간 후 대화창이 다시 투명해지며 사라진다(Fade-out). 단, AI가 답변을 출력 중일 때는 사라지지 않는다.

### 3.3. 아바타 조작 및 애니메이션
직관적인 마우스 조작을 통해 바탕화면에서의 활용도를 높인다.
* **드래그 앤 드롭:** 아바타를 마우스로 잡고 바탕화면 어느 곳이든 자유롭게 이동할 수 있어야 한다.
* **상호작용:** 클릭, 더블클릭 등의 이벤트 발생 시 아바타의 2D 이미지 또는 GIF 모션이 반응하도록 처리한다.
* **항상 위 (Always on Top):** 사용자가 다른 활성 창(크롬, ComfyUI 등)을 띄워도 아바타는 항상 최상단 화면에 노출되어야 한다.

### 3.4. 비동기 AI 통신 구조 (스레드 분리)
* UI 렌더링(PyQt5)과 Ollama API 통신 과정은 반드시 별도의 스레드(QThread)로 분리하여 비동기로 처리한다. 
* AI가 답변을 생성하는 동안 바탕화면 위젯이 멈추거나 튕기는 현상(프리징)이 절대 발생해서는 안 된다.

## 4. 필수 기술 요건
* `Qt.FramelessWindowHint` 및 `Qt.WA_TranslucentBackground` 속성을 사용하여 윈도우 테두리를 제거하고 완전한 투명 배경을 구현할 것.
* 복잡한 3D 엔진(Three.js, WebGL) 연동은 배제하고, 가볍고 빠릿한 PyQt5의 `QMovie`와 `QPixmap`을 활용한 2D/GIF 기반으로 설계할 것.
