# [Phase 5] 실시간 웹 검색 및 지식 증강(RAG) 통합 명세서

## 1. 프로젝트 목표
* **핵심 기능:** Ollama 로컬 모델의 오프라인 한계를 극복하기 위해, 실시간 인터넷 검색 결과(웹 크롤링)를 모델의 답변 생성 과정에 주입(Injection)함.
* **최적화 전략:** 별도의 유료 API 키가 필요 없는 무료 라이브러리를 사용하며, 네트워크 대기 시간 동안 아바타의 시각적 피드백을 강화함.

## 2. 기술 스택 및 라이브러리
* **검색 엔진:** `duckduckgo-search` (무료, 무제한, 익명성 보장)
* **데이터 파싱:** `BeautifulSoup4` (필요 시 웹 페이지 본문 정밀 추출)
* **비동기 처리:** `asyncio` 및 PyQt5 `QThread` (검색 중 UI 프리징 방지)

## 3. 상세 구현 로직 (RAG 워크플로우)

### 3.1. 검색 트리거링 (Search Intent Detection)
사용자의 입력 문장에서 실시간 정보가 필요한지 판단한다.
* **단순 키워드 매칭:** "오늘", "현재", "날씨", "시세", "뉴스", "검색해줘" 등의 키워드 포함 시 작동.
* **고급 판단:** Ollama에게 "이 문장이 최신 정보 검색이 필요한가?"를 짧게 물어보고 `True/False`를 반환받아 결정.

### 3.2. 실시간 데이터 수집 (Retrieval)
1. 사용자의 질문에서 핵심 검색어를 추출한다. (예: "오늘 파이코인 시세 알려줘" -> 검색어: "파이코인 시세")
2. `DDGS().text()` 함수를 사용하여 상위 3~5개의 검색 결과(제목 및 요약 문구)를 수집한다.
3. 수집된 텍스트 데이터에서 불필요한 광고 및 중복 문구를 필터링한다.

### 3.3. 컨텍스트 주입 (Prompt Injection)
수집된 정보를 포함하여 Ollama에게 보낼 최종 프롬프트를 재구성한다.
* **프롬프트 구조:**
  ```text
  [시스템 지시] 너는 검색 능력을 가진 AI 보좌관이다. 아래의 [검색 데이터]를 참고하여 답변하라.
  [검색 데이터] {검색된 웹 정보들...}
  [사용자 질문] {원본 질문}
  [답변 규칙] 반드시 이전 명세서에서 정의한 [감정 태그]를 문장 맨 앞에 붙여라.
