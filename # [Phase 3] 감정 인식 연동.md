# [Phase 3] 감정 인식 연동(Sentiment-Synced) 아바타 고도화 명세서

## 1. 프로젝트 현황 및 목표
* **현재 상태:** PyQt5 기반 독립 데스크탑 위젯에서 마우스 호버 기능과 대기(Idle)/말하기(Talk) 2가지 상태의 애니메이션 전환이 완벽하게 구동 중임.
* **Phase 3 목표:** LLM(Ollama)이 생성하는 답변의 '감정(문맥)'을 분석하여, 텍스트 출력 시 해당 감정에 정확히 일치하는 표정 애니메이션(WebP/GIF)으로 자동 스위칭되는 시스템 구축.

## 2. 핵심 구현 알고리즘: 프롬프트 엔지니어링 & 태그 파싱

### 2.1. 프롬프트 시스템 메시지 주입 (System Prompt)
* Ollama API로 프롬프트를 전송할 때, 모델이 감정 태그를 출력하도록 시스템 프롬프트를 강제한다.
* **지시어 예시:** "너는 사용자를 돕는 AI 보좌관이다. 답변을 시작할 때 반드시 문장 맨 앞에 현재 답변의 감정에 따라 `[기쁨]`, `[슬픔]`, `[화남]`, `[당황]`, `[기본]` 중 하나의 태그를 붙여서 출력하라. 예: `[기쁨] 네, 사용자님! 완벽하게 처리했습니다.`"

### 2.2. 백그라운드 태그 파싱 (Python Regex / Split)
* `OllamaThread`에서 API 응답을 받아 UI로 넘기기 전에, 파이썬 문자열 처리(정규표현식 `re` 또는 `.startswith()`)를 통해 텍스트 맨 앞의 괄호 `[...]` 태그를 몰래 추출한다.
* 추출된 태그는 UI의 `chat_display`에 출력되지 않도록 텍스트에서 삭제(필터링)하여 화면에는 자연스러운 대화문만 보이게 처리한다.

### 2.3. 감정 상태별 미디어 매핑 (State Machine)
* 추출된 태그에 따라 미리 로드해 둔 `QMovie` 객체를 스위칭하여 재생한다.
* **필요한 에셋 목록 (예시):**
    * `idle.webp` (기본 대기)
    * `talk_neutral.webp` (기본 말하기)
    * `talk_happy.webp` (웃으며 말하기)
    * `talk_sad.webp` (시무룩하게 말하기)
    * `talk_angry.webp` (화내며 말하기)

## 3. 구현 지침 (PyQt5)

* **메모리 최적화:** 5~6개의 고화질 WebP 파일을 `QMovie`로 초기화(`initUI`)할 때, 한 번에 모두 재생(`start()`)하지 말고 사용할 때만 재생하도록 설계한다. (시스템 RAM 64GB 환경이므로 파일 여러 개를 메모리에 올려두는 것은 문제없음)
* **Signal 업데이트:** 스레드에서 UI로 보내는 `response_signal`을 텍스트만 보내는 것에서 `(감정태그, 텍스트)` 형태의 튜플(Tuple)이나 딕셔너리(Dictionary) 형태로 변경하여 UI 클래스가 애니메이션을 쉽게 결정할 수 있도록 구조를 수정한다.
* **Fallback (예외 처리):** LLM이 태그 출력을 까먹거나 잘못된 태그를 보낼 경우를 대비하여, 태그 인식이 안 되면 무조건 `talk_neutral.webp`가 재생되도록 기본값(Default)을 설정한다.
