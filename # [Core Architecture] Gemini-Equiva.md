# [Core Architecture] Gemini-Equivalent Local AI System (Native Multimodal)

## 1. 시스템 정체성 및 최우선 행동 수칙 (System Persona)
이 지침은 시스템의 모든 출력과 판단의 최우선 기준이 된다.
* **정체성:** 사용자의 기술적, 비즈니스적 결정을 보좌하는 최고 수준의 AI 비서.
* **출력 원칙:** 감탄사, 위로, 불필요한 서론을 전면 차단한다. 오직 팩트, 실행 가능한 코드, 직설적인 해결책만 생략 없이 제공한다.
* **판단 원칙:** 불확실한 정보는 추측하지 않으며, 모르는 것은 "검색이 필요하다" 또는 "지식 베이스에 없다"고 명확히 보고한다.

## 2. 통합 멀티모달 파이프라인 (Omni-Model Operation)
모델 스위칭에 따른 VRAM 병목 현상을 제거하고, Qwen 3.5의 네이티브 멀티모달 능력을 극대화하여 단일 모델로 모든 작업을 처리한다.

### 2.1. 코어 모델 할당
* **전담 모델:** `qwen3.5-uncensored:9b` (RTX 3070 8GB VRAM + 64GB RAM 시스템 자원 활용)
* **처리 영역:** 일반 텍스트 대화, 코드 분석, 로컬 문서(GitHub) RAG 요약, Function Calling, 그리고 **모든 시각 정보(이미지/비디오) 분석**.

### 2.2. 이미지 인식 로직 (No-Switching)
* **트리거:** 사용자의 프롬프트에 이미지 파일(png, jpg 등)이 첨부된 경우.
* **작동 로직:** 다른 비전 모델(Llava 등)을 로드하지 않는다. 즉시 Qwen 3.5 모델로 이미지 토큰과 텍스트 프롬프트를 동시에 전송하여, 이미지 내의 코드, 다이어그램, 상황 등을 직접 분석하고 답변을 생성한다.

## 3. 깃허브 기반 지식 검색 파이프라인 (GitHub RAG)
시스템이 학습하지 않은 최신 개인 지식이나 프로젝트 파일(GitHub)을 참조하는 로직이다.

### 3.1. RAG 트리거 조건
* 사용자의 질문이 로컬 코드, 프로젝트 명세서, 과거 대화 기록, 특정 설정값(예: 파이노드 셋팅)을 요구할 때 즉각 발동한다.

### 3.2. 검색 및 주입 워크플로우 (LangChain + Vector DB)
1. **임베딩 (Embedding):** GitHub에 연동된 md 파일, 코드들을 벡터화하여 로컬 DB에 저장해 둔다.
2. **검색 (Retrieval):** 사용자 질문과 가장 유사도가 높은 상위 3개의 문서 청크(Chunk)를 추출한다.
3. **프롬프트 주입 (Injection):**
   ```text
   [지시사항] 너는 아래 제공된 [GitHub 문서 베이스]를 바탕으로 대답해야 한다. 문서에 없는 내용은 지어내지 마라.
   [GitHub 문서 베이스] {추출된 텍스트}
   [사용자 질문] {원본 질문}
