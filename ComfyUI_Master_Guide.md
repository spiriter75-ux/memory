# 🎨 ComfyUI & AI Video Generation 통합 가이드 (Master)

본 문서는 ComfyUI를 활용한 이미지/비디오 생성 모델(Wan, LTX 등)의 프롬프트 엔지니어링 및 워크플로우 통합 지식 베이스입니다.

---

## 1. Wan 2.2 I2V (Image-to-Video) 가이드
### 1.1. 완벽한 비디오 프롬프트의 6요소 (6-Element Framework)
1. **Subject (피사체)**: 장면에 등장하는 인물/사물의 구체적 정의.
2. **Action (동작)**: 피사체가 취하는 구체적인 움직임. (단순 동사보다 미시적 서술 권장)
3. **Setting (배경)**: 전경과 배경의 기하학적 심도 확립.
4. **Lighting (조명)**: 골든 아워, 림 라이트 등 분위기를 결정하는 광원 묘사.
5. **Camera (카메라)**: 렌즈 앵글, 화각, 이동 벡터 제어.
6. **Style (스타일)**: 최종 픽셀 렌더링 미학 및 질감(예: 16mm film grain).

### 1.2. MoE (Mixture-of-Experts) 아키텍처 이해
- **전문가 분할**: 초기 레이아웃을 잡는 '고노이즈 전문가'와 미세 디테일을 완성하는 '저노이즈 전문가'가 타임스텝($t_{moe}$)에 따라 교체됨.
- **VRAM 최적화**: 270억 개 파라미터 중 추론 시 140억 개만 활성화하여 RTX 3070 급에서도 운영 가능.

### 1.3. 고급 제어 기법 (CFG 1.0 & SVI)
- **긍정적 제약 (Positive Constraints)**: CFG 1.0 환경에서는 네거티브 프롬프트가 약하므로, "안 함" 대신 "무엇을 함"으로 대안 행동을 서사적으로 제공.
- **다단계 서사 (Progressive Prompting)**: 10초 이상의 롱폼 생성 시 도입->포커스->액션->절정 순으로 텍스트 인코더 개입을 시퀀스별로 분할.
- **환각 방지**: 등장인물 수량 엄격 고정(Cast Anchoring) 및 동작 세분화.

## 2. LTX 2.3 AI 비디오 생성 프롬프트 엔지니어링 및 제어
### 2.1. 프롬프트 핵심 작성 원칙 (Core Directives)
- **단일 흐름 단락**: 불렛 포인트 없이 하나의 자연스러운 단락으로 작성.
- **현재 시제 사용**: "동작한다", "움직인다" 등 생생한 현재 시제 동사 활용.
- **디테일 확장**: 10초 영상 기준 150~300 단어 수준의 상세 묘사 필요.
- **물리적 연기 지시**: 감정 단어 대신 눈물, 떨림, 시선 처리 등 물리적 행동으로 묘사.

### 2.2. 완벽한 프롬프트를 위한 6요소 구조
1. **샷 설정 (Shot)**: 크기(매크로, 클로즈업)와 장르 정의.
2. **장면 설정 (Scene)**: 조명(골든 아워, 림 라이트), 질감, 대기 분위기.
3. **액션 묘사 (Action)**: 시작부터 끝까지 일어나는 핵심 동작 서술.
4. **캐릭터 정의 (Character)**: 외양뿐만 아니라 신체적 연기(물리적 단서) 포함.
5. **카메라 워킹 (Camera)**: 돌리 인, 핸드헬드 트래킹 등 구체적 무빙 지시.
6. **오디오 묘사 (Audio)**: 앰비언트 노이즈, 발소리, 대사 등을 영상과 동기화하여 묘사.

### 2.3. 특수 워크플로우 (I2V & Portrait)
- **I2V (Image-to-Video)**: 이미지에 이미 있는 것을 설명하지 말고, **정적 상태가 깨지는 순간**과 모션의 전이에 집중.
- **세로형 비디오 (9:16)**: 모바일 환경에 최적화된 수직적 구도 강조.
- **부정적 제약**: 내면 상태 묘사 금지, 복잡한 텍스트/로고 생성 지양, 물리적 모순 방지.

## 3. Civitai & 외부 워크플로우 가이드 (lonecatone23 모델)
### 3.1. 워크플로우 모듈 구성
- **Prompt Assist**: 기본 입력 확장 및 보조.
- **ControlNet / I2I**: 레이아웃 고정 및 기존 이미지 변형.
- **Post-Production**: Hi-Rez Fix, Ultimate Upscalers, Face/Body Detailers를 통한 최종 출력 최적화.

### 3.2. 설치 및 노드 관리 (Troubleshooting)
- **누락 노드**: ComfyUI Manager의 'Install Missing Custom Nodes' 우선 실행. 해결 안 될 시 'Notes' 섹션의 GitHub 링크 확인.
- **GGUF 에러**: GGUF 모델 사용 시 "widget to string" 에러가 발생하면 위젯 이름을 `gguf-name`에서 다른 고유 이름으로 변경.
- **노드 업데이트**: 워크플로우와 커스텀 노드 버전을 항상 최신으로 동기화 필수.

### 3.3. 프롬프트 팁
- **중복 방지**: Prompt Assist 사용 시 메인 프롬프트에는 핵심 키워드만 배치하여 의도치 않은 열화(Deep-fried) 방지.
- **분리 입력**: 배경과 인물의 디테일을 분리하여 입력하는 것이 생성 품질 향상에 효과적.
