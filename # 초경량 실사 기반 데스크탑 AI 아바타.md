# [Standalone] 초경량 실사 기반 데스크탑 AI 아바타 개발 명세서

## 1. 프로젝트 목적
본 프로젝트는 고화질 실사 영상(AVI)의 핵심 구간을 추출하여, 시스템 자원(VRAM)을 최소한으로 사용하면서도 시각적으로는 매우 자연스럽고 부드러운 '바탕화면 상주형 AI 아바타'를 단독 프로그램으로 구축하는 것을 목표로 한다.

## 2. 시스템 자원 및 하드웨어 최적화
* **CPU/RAM:** Ryzen 7 5800H / 64GB RAM (넉넉한 시스템 메모리를 활용해 고화질 애니메이션 상주 가능)
* **GPU/VRAM:** RTX 3070 8GB (VRAM은 Ollama AI 추론에 우선 할당, 아바타는 경량화된 자원 사용)
* **AI 백엔드:** Ollama API (localhost:11434 / gemma-2-2b-it-gguf 등)

## 3. 아바타 애니메이션 제작 전략 (최소화/최적화)
실사 AVI 영상에서 다음 두 가지 핵심 구간만 추출하여 제작한다.

### 3.1. 대기 모션 (idle.gif / idle.webp)
* **추출 구간:** 입을 다물고 숨을 쉬거나 눈을 깜빡이는 1.5~2초 내외의 구간.
* **루프 방식:** '순방향 재생 + 역방향 재생'을 결합한 **핑퐁(Ping-Pong) 루프**를 적용하여 끊김 없이 무한 반복되도록 처리.
* **해상도:** 가로세로 400px 내외 (바탕화면 위젯용 최적 크기).

### 3.2. 대화 모션 (talk.gif / talk.webp)
* **추출 구간:** 자연스럽게 입을 뻐끔거리는 0.5~1초 내외의 짧은 구간.
* **작동 방식:** AI가 텍스트 답변을 출력하는 동안에만 무한 반복 재생되어 실제 말을 하는 듯한 착각을 부여함.

## 4. 핵심 UI/UX 기능 (사용자 조작)

### 4.1. 완벽한 배경 투명화 및 항상 위 (Always on Top)
* 캐릭터를 제외한 모든 영역은 투명하게 처리되어 바탕화면 아이콘이나 다른 작업 창 위에 자연스럽게 배치됨.
* `Qt.WindowStaysOnTopHint` 속성을 사용하여 어떤 작업을 하더라도 아바타가 가려지지 않음.

### 4.2. 마우스 휠 실사 크기 조절 (Zoom In/Out)
* 아바타 위에 마우스를 올리고 **휠(Wheel)**을 굴리면 캐릭터 크기가 즉시 조절됨.
* `wheelEvent`를 활용하여 위젯 전체 사이즈와 이미지 크기를 동기화(Scaling)함.

### 4.3. 호버(Hover) 기반 동적 인터페이스
* **평상시:** 아바타만 노출되어 영상의 움직임을 감상.
* **마우스 진입 시:** 대화 내역 창과 텍스트 입력창이 아바타 주변에 부드럽게 나타남.
* **마우스 이탈 시:** 대화가 중단된 상태면 자동으로 UI가 숨겨져 화면 가림을 최소화함.

## 5. 기술적 구현 핵심 (Python/PyQt5)
* **비동기 통신 (QThread):** Ollama와 통신하는 동안 아바타 영상이 멈추지 않도록 스레드를 분리함.
* **포맷 전환:** 실사 화질을 유지하기 위해 256색 GIF 대신 **알파 채널이 포함된 WebP** 사용을 권장함.
* **위젯 이동:** 마우스 왼쪽 버튼 드래그로 자유로운 위치 이동 보장.

## 6. 향후 관리 가이드
* **확장성:** 추후 다른 캐릭터 영상(AVI)이 생기면, 배경을 지운 2초 내외의 WebP 파일로 교체하는 것만으로 캐릭터 변경이 가능하도록 설계함.
