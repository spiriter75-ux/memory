1. 프로젝트 개요
소설이나 시나리오 텍스트를 입력하면 AI가 이를 분석하여 웹툰 제작을 위한 상세 스토리보드(콘티)를 추출해주는 웹 애플리케이션입니다. 이미지 생성 기능은 포함하지 않으며, 전문적인 이미지 생성 프롬프트와 연출 가이드를 텍스트로 제공하는 것에 집중합니다.
2. 핵심 기능 요구사항
A. 소설 입력 및 분석 (Editor)
텍스트 입력: 대용량 소설 텍스트를 붙여넣을 수 있는 텍스트 영역.
스토리보드 생성: 버튼 클릭 시 AI(Gemini API 등)를 호출하여 텍스트를 분석하고 구조화된 JSON 데이터로 변환.
결과 표시: 추출된 패널별로 [장면 설명, 이미지 프롬프트, 나레이션, 대사]를 카드 형태로 시각화.
B. 시리즈 바이블 (Series Bible) - 설정 관리
캐릭터(Characters): 이름, 외형 묘사, LoRA 트리거 단어 관리.
장소(Locations): 배경 명칭 및 상세 묘사 관리.
아이템/소품(Objects): 주요 소품이나 탈것의 특징 관리.
영속성: 입력된 설정은 브라우저(LocalStorage)에 저장되어 다음 작업 시에도 유지됨.
C. 제작 가이드 및 제약 사항 (Style & Constraints)
스타일 가이드: 지향하는 화풍(예: 현대 판타지, 로맨스 판타지 등) 설정.
전역 제약 사항: 모든 패널에서 지켜야 할 규칙(예: 우측통행, 특정 로고 제외 등) 설정.
화면 비율: 9:16(세로), 3:4(표준), 16:9(가로) 등 콘티의 기본 비율 설정.
D. 데이터 관리 (Data Management)
가져오기/내보내기: 작업 중인 스토리보드와 시리즈 바이블 설정을 JSON 파일로 저장하고 불러오기.
전체 초기화: 모든 데이터를 삭제하고 초기 상태로 되돌리는 기능.
3. 기술 스택 및 UI/UX 가이드라인
Framework: React (Vite), TypeScript.
Styling: Tailwind CSS (Modern, Clean, Minimalist 디자인).
Icons: Lucide React.
Animation: Framer Motion (부드러운 탭 전환 및 모달 효과).
UI 구성:
상단: 앱 이름 및 전체 초기화/내보내기 버튼이 있는 헤더.
중앙: '에디터(입력)'와 '시리즈 바이블(설정)'을 오가는 탭 시스템.
하단: 작업 진행 상태 및 도움말 정보.
4. AI 프롬프트 로직 (핵심 엔진)
AI에게 전달할 프롬프트는 다음 구조를 따라야 합니다:
역할 정의: "당신은 전문 웹툰 스토리보드 작가입니다."
입력 데이터: [소설 텍스트] + [시리즈 바이블 설정] + [스타일 가이드].
출력 형식: 반드시 다음 필드를 포함하는 JSON 객체여야 함.
title: 스토리 제목
characters: 분석된 캐릭터 리스트
panels: 패널 번호, 장면 묘사, 이미지 생성용 상세 프롬프트(영어), 나레이션, 대사.
특이사항: 입력값이 '슬라이드 목차' 형태일 경우 해당 구조를 엄격히 준수할 것.
5. 제외 사항 (Exclusion)
이미지 생성 API 연동: 실제 이미지를 생성하는 기능(Imagen, Stable Diffusion 등)은 구현하지 않음.
이미지 표시 영역: 이미지 대신 "이미지 프롬프트" 텍스트를 강조하여 표시하고 복사할 수 있는 버튼 제공.
